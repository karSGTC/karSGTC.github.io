系统环境：Ubuntu 22.04 + Linux 5.15 LTS
使用场景：Linux 内核编译 / ROS2 / 高负载开发

一、当前系统状态

完成系统修复与驱动安装后，系统已经达到一个**“可长期干活”的稳定状态**：

  • Linux 内核：5.15 LTS（稳定、兼容性最好）
  • NVIDIA 驱动已正确安装（nvidia-smi 正常）
  • 网络、输入设备全部恢复
  • 准备进入 高负载开发阶段

二、Linux 内核编译环境准备

必备依赖一次性装齐

关键命令解释

  • make menuconfig
    → 图形化配置内核功能
  • make -j$(nproc)
    → 使用全部 CPU 核心并行编译（ 极高发热）

易踩坑

  • 内核编译 极吃磁盘空间
  • 中间文件可达 15–30GB
  • 不建议在 128GB 系统盘直接编译

四、高负载核心问题：散热与风扇控制

六、NBFC 编译 / 安装失败的真实原因总结

make 报 “没有那个文件或目录”

常见原因：

  • 缺 -dev 结尾的开发包
  • 项目使用 cmake，却直接 make
  • 内核头文件未安装

补救：

用 deb 安装后，又手动 make 的问题

  • .deb 是 成品
  • make 是 现场加工
  • 两者混用极易冲突

要么 deb，要么源码，不要同时
