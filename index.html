拯救者Ubuntu自动安装显卡驱动后触摸板与网卡失效排错与修复

Ubuntu 22.04 + 联想拯救者 Y7000P（2019）

自动安装显卡驱动后触摸板 / Wi-Fi 全失效的完整排错与修复记录

关键词：Ubuntu 22.04、Lenovo Legion Y7000P、NVIDIA、HWE 内核、5.15、DKMS、触摸板失效、Wi-Fi 消失
适合人群：Linux 新手 / 嵌入式方向 / 双系统用户

一、问题背景

设备信息：

  • 笔记本：联想拯救者 Y7000P（2019）
  • 显卡：NVIDIA GTX 1650
  • 系统：Ubuntu 22.04
  • 安装选项：勾选了「自动安装第三方显卡驱动」

触发问题的操作

在 Ubuntu 安装阶段勾选了 自动化安装 NVIDIA 显卡驱动，系统安装完成后出现了以下严重问题：

  • 笔记本 触摸板完全失效
  • 内置 Wi-Fi 消失
  • 外接 USB Wi-Fi（RT3070）lsusb 能看到，但系统无无线选项
  • 手机 USB 共享网络无法识别
  • 系统已被自动升级到 6.8 HWE 内核

二、核心原因

「高版本内核 + NVIDIA DKMS + 笔记本硬件」冲突。

核心原因拆解

  1. Ubuntu 22.04 默认 GA 内核是 5.15
  2. 自动安装显卡驱动会：
    • 拉取 HWE 内核（6.8.x）
    • 同时触发 DKMS 重新编译内核模块
  3. 2019 款 Y7000P 的硬件（触摸板 i2c / Wi-Fi AC9560）：
    • 6.8 内核上存在兼容性断层
  4. DKMS 编译失败：
  • 显卡驱动没装好
  • 其他输入 / 网络模块被“挤掉”

6.8 内核 + NVIDIA 驱动在这台老机器上不稳定。

三、内核切换验证

进入 GRUB → Advanced options ：

  • 只有：
    • 6.8.0-94-generic
    • 6.8.0-40-generic
  • 没有 5.15

说明系统已经彻底迁移到 HWE 内核，单纯切换 6.8 的小版本 无法解决根因。

四、无网络状态排错

USB Wi-Fi（RT3070）

能看到 Ralink RT3070

但：

  • 没有 Wi-Fi 选项

  • iwconfig 无 wlan 接口

  • modprobe rt2800usb 报错：

结论：6.8 内核模块树已损坏 / 不完整

手机 USB 共享网络失败

尝试链路：

结果：

  • lsusb 能看到 Xiaomi RNDIS
  • ip link show 没有 usb0 / enp* 设备

结论：RNDIS 驱动在当前内核环境下未正确创建虚拟网卡

五、最终转折点

决策：放弃在线修复 Windows 离线安装内核

六、离线安装 Ubuntu 官方 5.15 内核

不用 mainline 内核

  • kernel.ubuntu.com/mainline 提供的是 原生社区内核

  • 缺少 Ubuntu 针对：

    • 笔记本 ACPI
    • 触摸板
    • 电源管理
      的适配补丁

用 Ubuntu 仓库的发行版内核（5.15.0-xxx）

Windows 下下载的文件清单

以 5.15.0-101 为例：

  • linux-image-5.15.0-101-generic_*.deb
  • linux-modules-5.15.0-101-generic_*.deb
  • linux-modules-extra-5.15.0-101-generic_*.deb

启动进入 5.15 后：

  • 触摸板恢复
  • 内置 Wi-Fi 恢复
  • RT3070 插上即用
  • 系统输入 / 网络完全正常

八、显卡驱动安装

错误做法

  • 自动安装
  • 乱点“附加驱动”
  • 安装最新版本（545 / 550 / 590）

正确流程

清理旧驱动残留

安装 NVIDIA 535

nvidia-dkms 决定驱动能否正确“缝合”到 5.15 内核

重启验证

项目 原因
内核 5.15 LTS
显卡 GTX 1650

DKMS 成功率高，不破坏其他模块

十、避坑清单

必避大坑

  • Ubuntu 安装时勾选自动显卡驱动（老机器）
  • HWE 内核无脑追新
  • 显卡驱动版本越新越好（错误）

正确习惯

  • 先保证 输入设备 + 网络
  • 能离线就离线修内核
  • 显卡驱动“够用就好”
  • 内核稳定

十一、关键知识点速查表

  • GA Kernel：5.15
  • HWE Kernel：6.x
  • DKMS：内核与驱动的桥梁
  • linux-modules-extra：外设驱动库
  • rt2800usb：RT3070 驱动模块
  • i2c_hid_acpi：笔记本触摸板模块

Ubuntu 22.04嵌入式开发环境搭建踩坑记录

设备：联想拯救者 Y7000P 2019

硬盘结构：

  • 1TB NVMe：Windows 10
  • 128GB SATA：Ubuntu 22.04（独立安装）

定位：Linux 嵌入式 / 交叉编译 / SoC BSP / Yocto

这篇文章不是教程，而是一份真实踩坑记录 + 原理复盘。如果你是嵌入式新手，或者第一次在笔记本上做 Win + Linux 物理双盘，这篇内容能帮你少浪费几天生命。

一、为什么我最终选择 Ubuntu 22.04 而不是 24.04

2026 时间点，嵌入式工程师默认首选 Ubuntu 22.04，除非你是全新项目 + 全新硬件。

1. 工具链与 SDK 现实问题

大量芯片原厂 SDK（NXP / TI / Rockchip / 海思 / 全志）：

  • 编译于 glibc 2.31 ~ 2.35
  • 假设宿主机是 Ubuntu 20.04 / 22.04

Ubuntu 24.04 使用 glibc 2.39，典型问题：

  • GLIBC_2.34 not found
  • 老交叉编译器直接无法运行

22.04 工业兼容性最佳”。

2. Yocto / BSP 支持现实

项目 推荐宿主机
Yocto Kirkstone (4.0 LTS) Ubuntu 22.04
老 BSP / Vendor SDK Ubuntu 22.04
Yocto Scarthgap (5.0) Ubuntu 24.04

结论：

优先 22.04，再考虑 24.04。

二、硬盘规划：为什么我把 Ubuntu 装到 128GB SATA

1. 物理隔离的好处

  • Windows 更新不会破坏 Linux 引导
  • Linux 折腾炸了，不影响 Win
  • GRUB 写坏 ≠ Windows 损坏

三、第一个大坑：联网安装 卡在“更新和其他软件”

表现

  • 卡 10 分钟
  • 卡 30 分钟
  • 看起来像死机

原因

1. 官方软件源在国外

安装器在后台跑：

默认连接 archive.ubuntu.com，在国内：

  • DNS 慢
  • TLS 卡
  • 直接超时

2. NVIDIA 驱动自动探测

勾选了：

  • “为图形和 Wi‑Fi 硬件安装第三方软件”

安装器会：

  • 访问 Canonical Partner 源
  • 匹配 GTX 显卡闭源驱动

3. 中文语言包阻塞下载

选了中文，安装器会试图一次性下载：

  • 中文 locale
  • 输入法
  • 字体

阻塞式下载。

正确解法（推荐）

断网安装

  • 拔网线 / 断 Wi‑Fi
  • 重新开始安装
  • 10–15 分钟完成

联网的事情，进系统再说。

四、大坑：安装结束时报 EFI 错误

报错核心含义

/boot/efi 看起来不像是一个 EFI 分区

人话：

没有给 GRUB 一个合法的 UEFI 启动位置。

EFI 分区

  • 文件系统：FAT32
  • 类型：EFI System Partition
  • 不能是 ext4
  • 不能只是挂载点叫 /boot/efi

五、分区方案推荐（128GB SATA）

在“安装类型”选择：其他选项 (Something else)

推荐结构

分区 大小 类型 用途
EFI 512MB FAT32 EFI System Partition
/ 100GB+ ext4 根文件系统
swap 8GB swap 编译防爆内存

细节

安装启动引导器的设备：请选择 /dev/sdX（整块盘）

❌ /dev/sda1

/dev/sda

六、失败（原理复盘）

  1. EFI 分区格式错误（ext4）
  2. 分区表不是 GPT
  3. 安装器想把 GRUB 写进 NVMe 的 Windows EFI
  4. SATA 盘根本没有 ESP

  • ❌ 不要一开始就联网安装
  • ❌ 不要自动分区
